Now we have the setup to test our Angular stuff in development. The main reason it wasn't working yesterday? You need to enclose all of your angular app logic in the ng-app tag--none of the small snippets/examples I looked through included this fact. Goes to show, you can't learn everything just by reading examples.

I wanted to spend some time building out an API doc for the services I'm building. Pydoc is a module that uses docstrings you write and generates an HTML doc file, so all I need to do is actaully write those docstrings. That also requires a little bit of refactoring the structure of my service.

Pydoc doesn't work as expected. Besides having difficulty generating docs for an entire package/module, the HTML it creates looks god-awful. There are a ton of other documentation generators out there that I will take a look at.

Next on the list after the global counter is a comment-section app, which will be followed by some sort of game (I cannot wait). On the infrastructure side, I want to create a 404 page and take the few minutes to set up nginx to point at the 404 page. So, those are the tasks for this week.
