Quite frankly, it looks like today's hour is mostly going to be reading. Our goal is to set up a comment section. I think a sub-goal might be to recreate the app from yesterday, but without using an external service. And I genuinely don't know, right now, where to start.

Five minutes of googling: We want to build a simple server/client app. I use ASP.NET at work, where Microsoft handles all of the nitty-gritty of both the hosting logic and the communication with the server. I don't want to use ASP.NET for this. Node seems to pop up a lot. Everything I know about node.js right now is it provides a server that interfaces easily with Javascript. The idea of using Node is kind of frightening, because why oh why would I want to write my back-end in Javascript? But, all signs point to this being a vital piece of web tech that I should definitely try to learn. Just need to verify that node will let me build out a global-counter type of thing...

I've verified that it should be possible, and I'm going into one of the most basic tutorials I've found. Godspeed.

Just to keep myself honest, we're twenty-seven minutes into the day and I'm still just reading. I've determined that Node.js can be used as a full server-client communication model, but as I somewhat expected, I don't have to write my backend in Javascript. The only thing I'm really interested for this specific task (the global counter) is AJAX, some way for the web page's script to update the page asynchronously. jQuery is the most popular Javascript library, although it's possible not all of these sites are using it for AJAX, but for the document navigation/event handling/solving problems posted on StackOverflow. The most common JavaScript frameworks, at least one of which I expect I'll be forced to use, AngularJS stands out as something I've seen people get excited about.

Thirty-seven minutes in and I'm reading Angular tutorials. It seems relatively convenient to work with, but I really do miss ASP.NET/Firebase/services that have everything set up conveniently for me. The best strategy I can come up with now, is to set up a separate web server just for services, with pages like "/getCommentSection?sectionId=blah" that do database read/writes. And I won't even get live updates, at least, not easily. I'll probably have to worry about security, firewalls, Tomcat--which, to be fair, I was going to do anyway--but having seen how easy it is to get servers running, I'm kind of wondering why anyone goes through LAMP anymore. Honestly, I'm strongly considering more and more just using Firebase.
